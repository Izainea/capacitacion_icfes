Fundamentos de CI/CD

El Puente entre el Desarrollo y las Operaciones

CI/CD son las siglas de Integración Continua y Entrega/Despliegue Continuo. Juntas, forman una filosofía y un conjunto de prácticas que automatizan las fases del ciclo de vida del software, permitiendo a los equipos entregar código de manera más frecuente y fiable.

Integración Continua (CI)

La Integración Continua es la práctica de fusionar (integrar) de forma automática y frecuente los cambios de código de múltiples desarrolladores en un repositorio central. Cada integración dispara un proceso automático de construcción y prueba.

Objetivos de la CI:

  • Detectar errores de integración de forma temprana.
  • Evitar que el código defectuoso llegue a ramas principales.
  • Reducir los conflictos de fusión.
  • Asegurar que la aplicación siempre esté en un estado "construible".
→ (Push a Git) → (Build) → (Test) → (OK)

Entrega y Despliegue Continuo (CD)

El CD lleva la automatización un paso más allá. Se refiere a dos prácticas relacionadas:

  • Entrega Continua (Continuous Delivery): Después de la fase de CI, el código se empaqueta y se despliega automáticamente en un entorno de "pre-producción" o "staging". El despliegue a producción final requiere una aprobación manual.
  • Despliegue Continuo (Continuous Deployment): Es el paso más avanzado. Cada cambio que pasa todas las pruebas se despliega automáticamente a producción, sin intervención humana.

Plataformas como Railway, Heroku o Netlify, y herramientas como GitHub Actions o Jenkins, son fundamentales para implementar pipelines de CI/CD.
