Más Allá del Reporte: Descargas Instantáneas

Los reportes parametrizados que vimos en la lección anterior son perfectos para crear documentos formales y completos. Sin embargo, a menudo los usuarios necesitan algo más simple y directo: la capacidad de descargar los datos con los que están trabajando o guardar una visualización específica como una imagen para usarla en una presentación.

Implementar funcionalidades de descarga directa es una de las formas más efectivas de aumentar el valor y la usabilidad de tu aplicación Shiny. Transforma al usuario de un simple espectador a un participante activo que puede llevarse los resultados de su exploración para usarlos en sus propias herramientas, como Excel o PowerPoint.

Taller 1: Descargar Datos Filtrados a CSV

Añadiremos un botón que permita al usuario descargar un archivo `.csv` que contenga exactamente los datos que está viendo en la tabla, respetando cualquier filtro que haya aplicado.

Paso 1: Añadir el Botón de Descarga a la UI

Al igual que con los reportes, usamos un downloadButton. Lo colocaremos en el `sidebarPanel` junto al botón de generación de reportes.

Acción: Modifica tu archivo `ui.R`.

# En ui.R, dentro del sidebarPanel(...)

      # ... (botón de descargar_reporte existente) ...
      
      downloadButton(
        outputId = "descargar_datos",
        label = "Descargar Datos (CSV)",
        icon = icon("file-csv")
      )

Paso 2: Implementar la Lógica de Descarga en el Server

Usamos nuevamente `downloadHandler`. La diferencia clave está en la función `content`. En lugar de renderizar un R Markdown, simplemente escribiremos el contenido de nuestro dataframe reactivo (`datos_filtrados()`) a un archivo CSV usando la función `write.csv()`.

Acción: Añade este bloque de código a tu `server.R`.

# En server.R, dentro de la función server(...)

  # --- Lógica para la descarga de datos CSV ---
  output$descargar_datos <- downloadHandler(
    
    # 1. Definimos el nombre del archivo
    filename = function() {
      paste0("datos-filtrados-icfes-", Sys.Date(), ".csv")
    },
    
    # 2. Definimos el contenido a escribir en el archivo
    content = function(file) {
      # La clave: usamos el mismo conductor reactivo que alimenta
      # nuestros gráficos y tablas.
      # Esto asegura que los datos descargados respeten los filtros.
      write.csv(datos_filtrados(), file, row.names = FALSE)
    }
  )

La Magia de la Reactividad

Observa cómo estamos reutilizando el conductor reactivo datos_filtrados(). Esta es la belleza de una buena arquitectura. La misma lógica que filtra los datos para el gráfico de dispersión y la tabla interactiva ahora también alimenta la función de descarga, garantizando consistencia total.

Taller 2: Guardar Gráfico como Imagen PNG

Ahora, permitiremos que el usuario guarde el gráfico de dispersión principal como un archivo de imagen de alta calidad.

Paso 1: Añadir el Botón en la UI

Colocaremos este botón directamente debajo del gráfico interactivo, en el `mainPanel`, para que esté contextualmente cerca de lo que se va a descargar.

Acción: Modifica tu `ui.R`.

# En ui.R, dentro del tabPanel("Gráfico Interactivo")

      # ... (plotlyOutput("scatter_plot") existente) ...
      
      hr(),
      downloadButton(
        outputId = "descargar_grafico",
        label = "Guardar Gráfico (PNG)",
        icon = icon("file-image")
      )

Paso 2: Implementar la Lógica de Guardado en el Server

El proceso es similar, pero en lugar de escribir datos, guardamos un gráfico. Para esto, usamos la función `ggsave()` de `ggplot2`. Un punto clave es que necesitamos **re-crear el objeto del gráfico** dentro del `downloadHandler` para poder pasárselo a `ggsave()`.

Acción: Añade este bloque de código a tu `server.R`.

# En server.R, dentro de la función server(...)

  # --- Lógica para guardar el gráfico ---
  output$descargar_grafico <- downloadHandler(
    filename = function() {
      paste0("grafico-dispersion-", Sys.Date(), ".png")
    },
    content = function(file) {
      # Creamos el objeto ggplot que queremos guardar.
      # Es el mismo código que está en renderPlotly, pero sin ggplotly().
      plot_para_guardar <- ggplot(datos_filtrados(), aes_string(x = input$var_x, y = input$var_y)) +
        geom_point(aes(color = cole_naturaleza), alpha = 0.6) +
        labs(
          title = paste("Relación entre", input$var_x, "y", input$var_y),
          x = input$var_x, y = input$var_y,
          color = "Naturaleza Colegio"
        ) +
        theme_minimal()
        
      # Usamos ggsave para guardar el gráfico en el archivo
      ggsave(file, plot = plot_para_guardar, device = "png", width = 10, height = 6, dpi = 300)
    }
  )
